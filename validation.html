Validation and Repeatability

The current demo checks scoped Boolean workflow models with a dependency-free bounded reference checker and an optional solver cross-check for supported examples.

What Has Been Validated

Reference Tests

Bounded reference checker tests pass for the included workflow examples.

Solver Cross-Check

The optional Z3 solver cross-check agrees with the bounded reference checker on supported examples when run in the local validated environment.

Repeatability Runs

Repeatability dry runs are completed across support, refund, permission, and data-export workflow families.

Trace and Retest Shape

Unsafe cases return reconstructable traces. Guarded variants return SAFE_WITHIN_BOUND under the supplied model and bound.

Why This Matters

The same audit pattern was applied across multiple workflow types without changing verifier source code. That supports the service workflow: intake, model, check, trace, remediate, and retest.

Limits of Validation

Validation shows repeatability on scoped Boolean workflow models. It does not establish unrestricted system safety or full real-world security coverage.
