# Do Not Share in First External Packet

## Purpose

This file defines what should remain internal during early external review of Track B.

## Do Not Include Initially

1. Track A / Lean formal proof package.
2. Unfinished Level 3 / TRACED_3_SAT broader proof work.
3. Internal RSVT/CRSL theory-heavy materials.
4. Current RSVT paper.
5. v0.3 implementation speculation beyond design docs.
6. `.venv` contents.
7. Python cache files.
8. Raw development scratch files.
9. Any claim implying unrestricted safety, deployment approval, or broad theorem completion.
10. Pricing/sales strategy unless explicitly preparing a business packet.
11. Customer-specific or sensitive data if real customer dry runs are later added.

## Why These Stay Internal

- Prevent claim inflation.
- Avoid confusing demo value with unfinished formal theorem work.
- Keep buyer-facing packet focused on bounded workflow audit value.
- Avoid exposing environment/dependency clutter.
- Avoid presenting internal strategy as public validation.

## Safe Public Subset

The public subset should focus on:

- one-page brief;
- buyer-facing value docs;
- public demo packet;
- repeatability validation matrix;
- selected examples;
- selected outputs;
- customer-style dry run;
- claim boundaries.

## Review Rule

Before sharing externally, check every file against:

- Is it necessary to understand bounded replay-risk audit value?
- Does it preserve bounded-scope language?
- Does it avoid unfinished theorem claims?
- Does it avoid internal strategy or implementation clutter?
